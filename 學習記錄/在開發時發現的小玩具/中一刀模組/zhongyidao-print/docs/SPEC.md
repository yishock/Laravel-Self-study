# 中一刀列印模組規格（跨專案）

> **性質**：獨立說明文件，不綁定任何 Git 專案或框架。  
> **用途**：記錄「中一刀」紙張、印表機、HTML/CSS 版型與**業務模組類型**；新專案實作時對照即可。  
> **存放**：本 repo 根目錄 `zhongyidao-print/`（與 ERP 專案分離，可 submodule 或複製）

---

## 1. 設計原則（給其他專案用）

1. **印表機列印範圍是上限**：`@page` 對齊驅動寬高，不要把洞區加進頁寬。  
2. **洞區只作排版留白**：用 padding / offset 避洞，不擴大 `@page`。  
3. **一種業務一個模組**：銷貨、出貨單等分開維護，共用「基礎 CSS 常數」即可。  
4. **常數集中**：紙張尺寸、每頁列數、品名折行規則放在一處（設定檔或共用樣式），避免每個模組各寫一套。  
5. **實機校正**：數字是起點，套印以尺量為準，用 `--print-offset-x/y`、`--pad-*` 微調。

---

## 2. 紙張與印表機（實際環境）

### 2.1 列印範圍 ≠ 物理紙全寬

驅動設定的寬高 = **印表機會印到的範圍**，**不含**左右打洞區（洞區）。

```
物理紙（示意）
┌─洞區─┬──── 列印範圍（驅動寬度）────┬─洞区─┐
│不計入│  ← 21.59 cm 全在這裡排版 →   │不計入│
└──────┴──────────────────────────────┴──────┘
```

### 2.2 Windows 自訂紙張「中一刀」

| 項目 | 值 |
|------|-----|
| 紙張名稱 | 中一刀 |
| 單位 | 公分 |
| 寬 (W) | **21.59** cm |
| 高 (H) | **13.97** cm |
| 邊界 L/R/T/B | **0.00**（皆無） |

| 換算 | 寬 | 高 |
|------|-----|-----|
| mm | **215.9** | **139.7** |
| inch（參考） | 8.5″ | 5.5″ |

**操作**：列印對話框選紙張「中一刀」、邊界無、縮放 **100%**、必要時勾背景圖形。

### 2.3 三層尺寸（開發時必分）

| 層級 | 建議尺寸 | 用途 |
|------|----------|------|
| **① 列印範圍** | 215.9 × 139.7 mm | `@page`、最外層 `.sheet-page` 寬高 |
| **② 表單框線區** | 196 × 140 mm | 套印單據主表格外框（在①之內置中或偏移） |
| **③ 洞區／四聯** | 不計入① | 僅 `--pad-left/right`、`--copy-gutter` 等留白 |

> 常見錯誤：把「24cm 含洞」設成 `@page` 寬（例如 240mm）。驅動不承認洞區寬度，易造成縮放或裁切。

## 3. 共用 CSS 基準（建議複製到新專案）

```css
:root {
    /* ① 列印範圍（對齊印表機） */
    --paper-w: 215.9mm;
    --paper-h: 139.7mm;

    /* ② 表單框（套印類模組） */
    --form-w: 196mm;
    --form-h: 140mm;

    /* 洞區／四聯（在①內留白，不加頁寬） */
    --pad-top: 5mm;
    --pad-bottom: 15mm;
    --pad-left: 10mm;      /* 依實印調，避左洞 */
    --pad-right: 10mm;     /* 依實印調，避右洞＋四聯 */
    --copy-gutter: 8mm;    /* 四聯直書字寬度（出貨單類） */

    /* 實機微調 */
    --print-offset-x: 0mm;
    --print-offset-y: 0mm;
}

@media print {
    @page {
        size: var(--paper-w) var(--paper-h);
        margin: 0;
    }
}
```

**字型（建議）**：Microsoft JhengHei／微軟正黑體；標題或套印欄可用標楷體。

## 4. 中一刀模組一覽（業務分類）

以下為**模組類型**清單，與語言、框架、檔名無關。新專案實作時自行命名檔案即可。

### 4.1 版型 A — 套印／全印單據（表單框 196×140）

適用：連續紙上已有框線，或整張列印含公司抬頭。結構多為**單頁一張**、表格式表頭＋明細＋合計。

| 模組代號 | 單據名稱 | 列印型態 | 備註 |
|----------|----------|----------|------|
| `sales-invoice` | 銷貨單 | 全印／套印／套印無額 | 可有「印抬頭／不印抬頭」變體 |
| `sales-return` | 銷退單 | 同銷貨單 | 版型與銷貨相近 |
| `purchase-order` | 採購單 | 全印 | |
| `goods-receipt` | 進貨單 | 全印 | |
| `stock-adjustment` | 庫存調整單 | 全印 | 字級可略大（如 16px） |
| `stock-transfer` | 庫存調撥單 | 全印 | 同調整單結構 |

**列印型態說明**

| 型態 | 說明 |
|------|------|
| 全印 | 含抬頭、框線、金額，印於空白紙或自印表單 |
| 套印 | 對準預印框線，只印變動欄位；可選是否印公司抬頭 |
| 套印無額 | 套印但不印金額欄 |
| 批次 | 多張單據一次開啟列印，每張 `page-break-after: always` |

**版型 A — 建議技術參數**

| 項目 | 建議值 |
|------|--------|
| `@page` | 196×140 mm **或** 215.9×139.7 mm（外頁大、內框 196 置中；二擇一策略，團隊統一即可） |
| 外層容器 | 寬高與表單框一致 |
| `html` 字級 | 14px |
| 表身列高 | 約 19–20px |
| 明細分頁 | 表身「列數單位」上限 **12**；品名折行：每行 **12** 字寬（中文=1、英數=0.5） |
| 換頁 | 超過上限換頁；不足頁補空白列 |

**客戶列印方式（選項型，非必備）**

若系統要讓客戶選預設列印型態，可對照：

| 選項 | 行為 |
|------|------|
| 列印銷貨單 | 全印＋抬頭 |
| 列印銷貨單(不印抬頭) | 全印無抬頭 |
| 套印銷貨單 | 套印＋抬頭 |
| 套印銷貨單(不印抬頭) | 套印無抬頭 |
| 套印無額銷貨單 | 套印不印金額 |
| 套印無額銷貨單(不印抬頭) | 同上且不印抬頭 |

### 4.2 版型 B — 自排版出貨單（四聯＋主表）

適用：**產品出貨單／訂購憑單**等需自畫框線、右側四聯直書、主表多欄。

| 模組代號 | 單據名稱 | 列印型態 | 備註 |
|----------|----------|----------|------|
| `shipment-order` | 產品出貨單（訂購憑單） | 全印 | 可參考 Excel 版型（如 12 欄 × 18 列概念） |

**版型 B — 建議技術參數**

| 項目 | 建議值 |
|------|--------|
| `@page` | **215.9 × 139.7 mm**（對齊 §2.2，勿用 240mm） |
| 主表欄數 | 12 欄（比例見下） |
| 明細每頁列數 | **8** 列（字級放大時由 10 列下修） |
| 品名過長 | 每列最多 **32** 字元後拆續列 |
| 四聯側 | `--copy-gutter: 8mm`，直書，在列印範圍內 |

| 表尾列 | 7 |

| 簽章列 | 7 |

**建議字級（px）**

## 5. 不屬於中一刀的報表（避免誤用）

下列通常為 **A4 直向** 或螢幕分頁，**不要**套用中一刀 `@page`：

- 運費／帳款／財務類長表報表  
- 傳票、損益表、資產負債表等  

若註解寫「支援中一刀」，需另開模組，不可與 A4 共用同一 `@page`。

## 6. 新專案接入步驟

1. 印表機建立自訂紙「中一刀」21.59×13.97 cm，邊界 0。  
2. 複製 §3 的 `:root` 與 `@media print`。  
3. 依 §4 勾選需要的**模組代號**，每模組一個 HTML 範本（或元件）。  
4. 實印一張，尺量框線與洞距，調 `--pad-*`、`--print-offset-*`。  
5. 明細分頁邏輯抽成共用函式（列數上限、品名拆行）。  
6. 文件只更新本規格；**勿**把路徑寫進本檔。

## 7. 新模組檢查清單

- [ ] 已確認屬於版型 A 或 B  
- [ ] `@page` = 215.9×139.7 mm（或團隊約定之統一策略）  
- [ ] 洞區未加進 `@page` 寬度  
- [ ] 表單框 196 mm 落在列印範圍內  
- [ ] 瀏覽器：中一刀、100%、無邊界  
- [ ] 分頁常數已集中、可設定  
- [ ] 多頁時最後一頁 `page-break-after: auto`  
- [ ] 已實機套印通過並記錄校正後的 pad/offset 數值（可寫在專案自己的 README，不寫進本檔）
